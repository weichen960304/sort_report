### 氣泡排序法 (Bubble Sort)

*   **簡述原理：**
    氣泡排序法是一種簡單的排序演算法。它會重複地走訪過要排序的數列，一次比較兩個相鄰的元素，如果它們的順序錯誤就把它們交換過來。走訪數列的工作是重複地進行直到沒有再需要交換，也就是說該數列已經排序完成。越大的元素會經由交換慢慢「浮」到數列的頂端。
*   **複雜度分析：**
    *   最佳時間複雜度：$O(n)$ （當資料已經是排序好的狀態，只需走訪一次）
    *   平均時間複雜度：$O(n^2)$
    *   最壞時間複雜度：$O(n^2)$
    *   空間複雜度：$O(1)$ （只需一個額外的空間用於交換）
*   **模擬實作內容：**
    實作上通常使用雙重迴圈。外層迴圈控制需要走訪的趟數（共 $n-1$ 趟），內層迴圈則負責比較相鄰元素並執行交換。為了優化，可以加入一個布林變數（flag）來記錄該趟走訪是否發生過交換，若某一趟完全沒有交換，代表陣列已排序完成，可提早結束迴圈。

### 選擇排序法 (Selection Sort)

*   **簡述原理：**
    選擇排序法的概念是將陣列分為「已排序」與「未排序」兩個部分。初始狀態下，已排序部分為空。演算法會不斷從未排序的部分中尋找最小值（或最大值），然後將其與未排序部分的第一個元素交換位置，藉此將已排序部分的長度逐步增加，直到全部排序完成。
*   **複雜度分析：**
    *   最佳時間複雜度：$O(n^2)$
    *   平均時間複雜度：$O(n^2)$
    *   最壞時間複雜度：$O(n^2)$ （無論資料初始狀態為何，都需要進行相同次數的比較）
    *   空間複雜度：$O(1)$
*   **模擬實作內容：**
    同樣採用雙重迴圈結構。外層迴圈用於逐步推進已排序序列的邊界，內層迴圈則用於在剩下的未排序陣列中尋找最小值的索引（index）。當內層迴圈結束找到最小值的索引後，再將該元素與未排序區間的第一個元素進行位置互換。

### 插入排序法 (Insertion Sort)

*   **簡述原理：**
    插入排序法如同我們打撲克牌時整理手牌的方式。它會將陣列分為「已排序」和「未排序」兩部分。每次從未排序部分取出一個元素，由後往前掃描已排序的部分，找到合適的位置將其「插入」，直到所有未排序元素都處理完畢。
*   **複雜度分析：**
    *   最佳時間複雜度：$O(n)$ （資料已排序時，每次只需比較一次）
    *   平均時間複雜度：$O(n^2)$
    *   最壞時間複雜度：$O(n^2)$
    *   空間複雜度：$O(1)$
*   **模擬實作內容：**
    實作時，外層迴圈從索引 1 開始往後走訪（假設索引 0 已經預設為排好序的單一元素）。內層迴圈則將當前取出的指標元素與前面已排序的元素進行比較，若前面的元素較大，則將該元素往後平移一個位置，直到找到指標元素應該放入的正確位置為止。

### 合併排序法 (Merge Sort)

*   **簡述原理：**
    合併排序法採用了「分而治之 (Divide and Conquer)」的策略。它將原始陣列不斷從中間分割成兩半，直到每個子陣列只剩下一個元素為止（此時視為已排序）。接著，再將這些小的子陣列兩兩合併成較大的已排序陣列，層層遞迴向上合併，最終得到完全排序的陣列。
*   **複雜度分析：**
    *   最佳時間複雜度：$O(n \log n)$
    *   平均時間複雜度：$O(n \log n)$
    *   最壞時間複雜度：$O(n \log n)$
    *   空間複雜度：$O(n)$ （在合併的過程中需要額外的陣列空間來暫存資料）
*   **模擬實作內容：**
    實作分為兩個主要函式：一個是負責「分割」的遞迴函式，計算中間點並將左右兩半再次傳入自身；另一個是負責「合併」的函式，需要宣告一個暫存陣列，並使用兩個指標分別指向左右兩個已排序子陣列的開頭，比較大小後依序填入暫存陣列，最後再將暫存陣列的內容複製回原陣列。

### 快速排序法 (Quick Sort)  

*   **簡述原理：**
    快速排序法同樣使用「分而治之」的概念。它會在陣列中挑選一個基準值（Pivot），然後將陣列重新排列：所有比基準值小的元素放到基準值左邊，比基準值大的放到右邊。這個動作稱為分區（Partition）。接著對左右兩個子陣列遞迴執行相同的操作，直到子陣列長度為 1 或 0。
*   **複雜度分析：**
    *   最佳時間複雜度：$O(n \log n)$ （每次都能均勻分割陣列）
    *   平均時間複雜度：$O(n \log n)$
    *   最壞時間複雜度：$O(n^2)$ （當資料已排序且選擇到極端值作為 Pivot 時）
    *   空間複雜度：$O(\log n)$ （遞迴呼叫的堆疊空間，最壞情況下為 $O(n)$）
*   **模擬實作內容：**
    透過遞迴實作。核心的 `partition` 函式通常會選擇最右邊或最左邊的元素作為 Pivot。透過兩個指標（一個記錄小於 Pivot 的邊界，一個用於走訪陣列），將小於 Pivot 的元素交換到左側，最後將 Pivot 放到左右區間的分界點上，並回傳該分界點的索引，供後續遞迴呼叫使用。

### 綜合比較

| 排序演算法 | 平均時間複雜度 | 最壞時間複雜度 | 最佳時間複雜度 | 空間複雜度 | 穩定性 (Stability) |
| :--- | :---: | :---: | :---: | :---: | :---: |
| **氣泡排序** | $O(n^2)$ | $O(n^2)$ | $O(n)$ | $O(1)$ | 穩定 |
| **選擇排序** | $O(n^2)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 不穩定 |
| **插入排序** | $O(n^2)$ | $O(n^2)$ | $O(n)$ | $O(1)$ | 穩定 |
| **合併排序** | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ | 穩定 |
| **快速排序** | $O(n \log n)$ | $O(n^2)$ | $O(n \log n)$ | $O(\log n)$ | 不穩定 |

### 學習心得

在這次的排序演算法模擬與實作中，我深刻體會到不同演算法在設計邏輯上的差異及其帶來的效能影響。氣泡、選擇與插入排序法的邏輯較為直觀且易於實作，對於小規模的資料集來說很方便，且具有空間複雜度 $O(1)$ 的優勢。然而，當資料量變大時，它們 $O(n^2)$ 的時間複雜度會導致執行時間大幅增加。

相較之下，採用「分而治之」策略的合併排序與快速排序，雖然實作上需要處理遞迴邏輯與指標邊界，較為複雜，但在處理龐大資料時，$O(n \log n)$ 的效能表現遠遠優於基礎排序法。透過這次報告，你不僅熟悉了程式語言對於陣列與記憶體的操作，也學習到在實際開發中，必須根據「資料規模」、「是否需要穩定排序」以及「記憶體空間限制」來權衡並選擇最適合的演算法。
